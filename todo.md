# 정보처리기사 실기 대비 앱 보강 Todo

## 기준 문서

- `information_processing_engineer_practical_exam_strategy.md`
- 이 문서는 최근 복원 문제 흐름, 언어별 함정, 예상문제 생성 규칙, 오답노트 작성 형식을 정리한 앱 보강 기준 문서다.

## 제품 방향

- 어디서든 접속해 정보처리기사 실기 시험을 대비할 수 있는 웹 학습 도구로 발전시킨다.
- 단순 기출 풀이 앱에서 기출, 개념 암기, 실전 변형 문제, 오답 반복을 연결하는 학습 앱으로 확장한다.
- 핵심 학습 흐름은 `기출 풀이 -> 해설 이해 -> 관련 개념 카드 복습 -> 유사 실전 문제 -> 오답 유형 정리`다.

## 전략 문서 분석 요약

- 최근 실기는 단일 문법보다 여러 개념을 결합한 문제 비중이 높다.
- Python 비중과 복합도가 증가하고 있어 Python, SQL/DB, 이론 단답을 안정 점수 확보 영역으로 둔다.
- C는 포인터, 구조체, 출력 형식, 함수 포인터 등 고난도 함정이 많으므로 단계별 추적 해설이 중요하다.
- Java는 참조 타입과 실제 객체 타입, 오버로딩과 오버라이딩 구분이 핵심이다.
- SQL은 논리 처리 순서와 중간 결과표가 중요하다.
- 비프로그래밍 이론은 긴 설명보다 핵심 키워드, 구분 규칙, 시험 신호를 빠르게 반복해야 한다.
- 오답노트는 문제 원문보다 `틀린 이유`, `정확한 규칙`, `문제에서 보인 신호`, `다음 풀이 행동` 중심으로 쌓는다.

## 1차 구현 목표

- 개념 카드 학습 섹션을 추가한다.
- 프로그래밍 외 이론뿐 아니라 Python/C/Java/SQL의 대표 함정도 카드로 빠르게 복습할 수 있게 한다.
- 섹션별 카드 진행률을 저장한다.
- 카드별 상태는 `알고 있음`, `헷갈림`, `모름`으로 기록한다.
- 홈에서 기출 풀이와 개념 카드 복습으로 자연스럽게 이동하게 한다.

## 개념 카드 섹션

- Python 함정
  - 중첩 리스트
  - 얕은 복사
  - `append`와 `extend`
  - 슬라이싱
  - 가변 기본 매개변수
  - `and`, `or` 반환값
- C 함정
  - 배열과 포인터
  - 구조체 포인터
  - 문자와 문자열
  - `printf` 출력 형식
  - 비트 연산
  - 함수 포인터
- Java 함정
  - 업캐스팅
  - 오버로딩과 오버라이딩
  - 생성자 실행 순서
  - 필드와 메서드 바인딩 차이
  - `static`
  - 예외 처리
- SQL/DB
  - `COUNT(*)`와 `COUNT(column)`
  - `DISTINCT`
  - `WHERE`와 `HAVING`
  - SQL 논리 처리 순서
  - `NULL`
  - 정규화와 ACID
- 네트워크/보안
  - ARP/RARP
  - CIDR 블록 크기
  - SQL Injection
  - XSS
  - SSRF
  - 워터링 홀
- 소프트웨어 공학
  - 스텁과 드라이버
  - 응집도 순서
  - 결합도 순서
  - 테스트 커버리지
  - 디자인 패턴 핵심 키워드
