# 4. 프로그래밍 문제 공통 출제 원칙

프로그래밍 문제는 단순 문법 확인보다 다음 능력을 평가하도록 구성한다.

1. 변수의 현재 값 추적
2. 변수의 자료형 추적
3. 참조 또는 포인터가 가리키는 대상 추적
4. 반복문 실행 횟수 계산
5. 함수 호출 순서 파악
6. 오버로딩과 오버라이딩 구분
7. 얕은 복사와 객체 공유 여부 확인
8. 출력 형식 적용
9. 연산자 우선순위 해석
10. 최종 출력값 계산

문제는 지나치게 비현실적인 난독화 코드만 만들지 말고, 실제 기출에서 볼 수 있는 수준의 함정을 사용한다.

좋은 문제의 기준:

- 핵심 함정이 1~3개 존재한다.
- 정답은 명확하다.
- 언어 명세상 모호하거나 실행 환경에 따라 결과가 달라지는 코드는 피한다.
- 정의되지 않은 동작을 정답 근거로 사용하지 않는다.
- 단순 암산보다 코드 추적이 필요하다.
- 해설에서 각 줄의 변화와 자료형을 설명할 수 있어야 한다.

## 5.2 Python 대표 함정

### 함정 1. 인덱스의 값이 숫자가 아니라 리스트

```python
a = [[1], [2], [3]]
```

여기서:

```python
a[1]
```

의 값은 `2`가 아니라 `[2]`다.

따라서:

```python
a[1] += a[0]
```

은 다음과 같다.

```python
[2] += [1]
```

결과:

```python
[2, 1]
```

예상문제 생성 시 중첩 리스트를 이용해 수험생이 인덱스 값의 자료형을 추적하도록 한다.

---

### 함정 2. 리스트의 `+`와 `+=`

```python
a = [1, 2]
b = [3, 4]
```

```python
a + b
```

는 새 리스트를 만든다.

```python
a += b
```

는 기존 리스트를 확장하는 방식으로 동작한다.

얕은 복사와 함께 나오면 원본 리스트 내부 객체도 변경될 수 있다.

---

### 함정 3. `append`와 `extend`

```python
a = [1, 2]
a.append([3, 4])
```

결과:

```python
[1, 2, [3, 4]]
```

```python
a = [1, 2]
a.extend([3, 4])
```

결과:

```python
[1, 2, 3, 4]
```

예상문제에서는 두 메서드의 결과 차이를 묻거나 중첩 리스트 상태를 계산하도록 한다.

---

### 함정 4. 얕은 복사

다음은 모두 기본적으로 얕은 복사다.

```python
b = a[:]
b = list(a)
b = a.copy()
```

중첩 리스트에서는 바깥 리스트만 새로 만들어지고 내부 객체는 공유된다.

```python
a = [[1], [2]]
b = a[:]
b[0].append(3)
```

최종적으로 `a[0]`과 `b[0]`은 모두 `[1, 3]`이 된다.

예상문제에서는 반드시 참조 공유 여부를 추적하게 한다.

---

### 함정 5. 슬라이싱

기본 형식:

```python
a[start:end:step]
```

중요 규칙:

- `end`는 포함되지 않는다.
- `step`이 음수이면 역방향으로 이동한다.

필수 패턴:

```python
a[:]       # 전체
a[::-1]    # 역순
a[::-2]    # 뒤에서부터 두 칸씩
a[::2]     # 0, 2, 4 ...
a[1::2]    # 1, 3, 5 ...
a[1:4]     # 인덱스 1, 2, 3
```

예상문제에서는 리스트뿐 아니라 문자열에도 슬라이싱을 적용한다.


문제를 생성할 때 같은 연산자가 자료형에 따라 다르게 동작하도록 구성할 수 있다.

### 함정 7. `range`

```python
range(1, 4)
```

의 값은:

```text
1, 2, 3
```

끝값은 포함되지 않는다.

다음 유형을 포함한다.

```python
range(n)
range(1, n)
range(0, n, 2)
range(n, 0, -1)
```

### 함정 8. 문자열 함수

```python

```


### 함정 9. 딕셔너리 키 중복

```python

```

```python

```

### 함정 10. 가변 기본 매개변수

```python

    a.append(1)
    return a
```

연속 호출 시 기본 리스트가 재사용된다.

```python
print(f())
print(f())
```

출력:

```text

[1, 1]
```

이 유형은 중상 난도로 제한적으로 출제한다.

---

### 함정 11. `and`, `or`의 반환값

Python의 `and`, `or`는 항상 Boolean을 반환하지 않는다.

```python
0 or 10
# 10
```

```python
5 and 10
# 10
```

참과 거짓 여부를 판단한 뒤 피연산자 자체를 반환할 수 있다.

## 6.2 C 대표 함정

### 함정 1. 배열과 포인터

```c
p[i] == *(p + i)
```

```c
&p[i] == p + i
```

`int*` 포인터에 `+1`을 하면 다음 `int` 요소의 주소로 이동한다.

```c
int n[] = {16, 32};
int *p = n;
```

```c
p       // &n[0]
p + 1   // &n[1]
*(p+1)  // 32
```

---

### 함정 2. 역참조와 함수 호출 순서

```c
*fn(arr)
```

은 다음 순서로 해석한다.

1. `fn(arr)` 호출
2. 주소 반환
3. `*`로 해당 주소의 값 확인

---

### 함정 3. 함수 포인터

```c
int* (*fn)(int*);
```

해석:

- `fn`은 함수 포인터
- 매개변수는 `int*`
- 반환값은 `int*`

예상문제에서는 함수 포인터 선언을 직접 작성하게 하기보다 호출 결과를 해석하도록 한다.

---

### 함정 4. 구조체

```c
struct Item {
    int value;
};
```

일반 구조체 변수:

```c
item.value
```

구조체 포인터:

```c
p->value
```

`p->value`는 `(*p).value`와 같다.

---

### 함정 5. 문자와 문자열

```c
'A'
```

는 문자 하나다.

```c
"A"
```

는 문자열이다.

문자 연산은 문자 코드값을 기준으로 이루어진다.

```c
'D' - 'A'
```

결과는 `3`이다.

|---|---|

### 함정 7. 증감 연산자


- 함수형 인터페이스와 람다

---

## 7.2 Java 대표 함정

### 함정 1. 업캐스팅

```java
A a = new B();
```

해석:

```text
참조 변수 타입: A
실제 객체 타입: B
```

수험생은 항상 두 타입을 따로 기록해야 한다.

---

### 함정 2. 오버로딩과 오버라이딩

오버로딩:

```java
f(Object x)
f(String x)
```

메서드 이름은 같지만 매개변수 타입이 다르다.

오버라이딩:

```java
class A {
    String f(Object x) {
        return "A";
    }
}

class B extends A {
    @Override
    String f(Object x) {
        return "B";
    }
}
```

부모의 같은 메서드를 자식이 재정의한다.

핵심 규칙:

```text
오버로딩 후보 선택: 참조 변수 또는 컴파일 시점 타입 기준
오버라이딩 실제 실행: 실제 객체 타입 기준
```

---

### 함정 3. 참조 타입이 사용할 수 있는 범위

```java
A a = new B();
```

`a`는 `A`에 선언된 멤버를 기준으로 호출 가능 여부가 결정된다.

`B`에만 존재하는 메서드는 직접 호출할 수 없다.

---

### 함정 4. 생성자 실행 순서

기본 흐름:

```text
부모 클래스 초기화
→ 부모 생성자
→ 자식 클래스 초기화
→ 자식 생성자
```

`static` 초기화 블록이 있으면 클래스 최초 로딩 시 먼저 실행된다.

---

### 함정 5. 부모 생성자에서 오버라이딩 메서드 호출

부모 생성자 안에서 오버라이딩 가능한 메서드를 호출하면 자식 클래스의 메서드가 실행될 수 있다.

하지만 자식 필드 초기화가 완료되기 전일 수 있으므로 기본값이 출력될 수 있다.

이 유형은 중상 난도로 출제한다.

---

### 함정 6. 필드와 메서드의 차이

메서드는 동적 바인딩이 적용된다.

필드는 참조 변수 타입에 따라 선택된다.

```java
Parent p = new Child();
```

```java
p.value
```

는 `Parent`의 필드가 선택될 수 있다.

```java
p.show()
```

는 `Child`의 오버라이딩 메서드가 실행될 수 있다.

---

### 함정 7. 문자열 연결

```java
9 + 2 + "2" + 3
```

왼쪽부터 계산한다.

```text
9 + 2 = 11
11 + "2" = "112"
"112" + 3 = "1123"
```

문자열이 등장한 이후에는 문자열 연결이 된다.

---

### 함정 8. 예외 처리

```java
try
catch
finally
```

`catch`가 실행되어도 일반적으로 `finally`는 실행된다.

예상문제에서는 예외 발생 위치와 이후 문장 실행 여부를 묻는다.

---

### 함정 9. `static`

`static` 필드는 객체별이 아니라 클래스 단위로 공유된다.

여러 객체를 생성하면서 값이 누적되는 문제를 출제할 수 있다.

## 8.2 SQL 대표 함정

### 함정 1. `COUNT(*)`와 `COUNT(column)`

```sql
COUNT(*)
```

모든 행을 센다.

```sql
COUNT(column)
```

해당 열이 `NULL`이 아닌 행만 센다.

---

### 함정 2. `DISTINCT`

```sql
SELECT DISTINCT a, b
```

`a`만 중복 제거하는 것이 아니라 `(a, b)` 조합의 중복을 제거한다.

---

### 함정 3. `WHERE`와 `HAVING`

```text
WHERE: 그룹화 전 행 조건
HAVING: 그룹화 후 그룹 조건
```

---

### 함정 4. SQL 논리 처리 순서

문제를 풀 때 다음 순서로 추적한다.

```text
FROM / JOIN
→ WHERE
→ GROUP BY
→ 집계 함수
→ HAVING
→ SELECT
→ DISTINCT
→ ORDER BY
```

---

### 함정 5. `NULL`

```sql
column = NULL
```

은 올바른 비교가 아니다.

```sql
column IS NULL
```

을 사용한다.

---

### 함정 6. 조인 결과 행 증가

조인 키가 유일하지 않으면 결과 행이 예상보다 증가할 수 있다.

수험생이 직접 중간 결과표를 그리도록 문제를 구성한다.

---

### 함정 7. 서브쿼리 경계값

```sql
score > (SELECT AVG(score) ...)
```

와:

```sql
score >= (SELECT AVG(score) ...)
```

를 구분한다.

## 15.2 프로그래밍 문제 해설 형식

프로그래밍 문제는 반드시 다음 순서로 해설한다.

1. 초기 변수 값
2. 변수 자료형
3. 함수 호출 또는 반복문 흐름
4. 각 단계의 값 변화
5. 참조 또는 포인터 공유 관계
6. 최종 계산값
7. 출력 형식 적용
8. 정답

반복문이 있다면 표를 사용한다.

예시:

| 반복 | 실행문 | 변경 결과 |
|---:|---|---|
| 0 | `b[1] += b[0]` | `[2, 1]` |
| 1 | `b[2] += b[1]` | `[3, 2, 1]` |

## 15.3 문제 생성 시 금지사항

- 정의되지 않은 동작에 의존하는 C 코드
- 실행 환경마다 결과가 달라질 수 있는 코드
- 라이브러리 버전에 따라 결과가 달라지는 코드
- 지나치게 긴 코드
- 단순히 문법을 외웠는지만 묻는 문제만 반복
- 실제 시험 범위와 지나치게 동떨어진 알고리즘
- 정답이 여러 개 가능한 모호한 단답형
- 문제 설명과 정답이 일치하지 않는 코드

# 17. 오답 노트 작성 형식

문제 자체를 그대로 베끼지 말고 함정 유형별로 기록한다.

```text
[언어/영역]

틀린 이유:
내가 어떤 값을 잘못 해석했는가?

정확한 규칙:
언어 또는 개념의 실제 규칙은 무엇인가?

문제에서 보인 신호:
어떤 코드나 표현을 보고 함정을 알아차려야 하는가?

다음 풀이 행동:
다음에는 무엇을 먼저 적어야 하는가?
```

예시:

```text
[Python - 중첩 리스트]

틀린 이유:
b[i]를 숫자로 생각했다.

정확한 규칙:
b = [[1], [2]]이면 b[i]는 list다.

문제에서 보인 신호:
리스트 컴프리헨션의 결과가 [x] 형태였다.

다음 풀이 행동:
반복문 전에 b[0], b[1]의 실제 값과 자료형을 적는다.
```

# 18. 최종 학습 전략

1. Python 문법을 먼저 체계적으로 정리한다.
2. Python 문제는 매 줄마다 값과 자료형을 적는다.
3. SQL은 중간 결과표를 그리는 연습을 한다.
4. Java는 참조 타입과 실제 객체 타입을 분리해서 기록한다.
5. C는 배열과 포인터부터 시작한다.
6. 함수 포인터와 이중 포인터는 기본 포인터 이후에 공부한다.
7. 이론은 긴 설명보다 핵심 키워드로 암기한다.
8. 오래된 기출은 기초 문법 훈련용으로 사용한다.
9. 최근 기출은 복합 함정 적응용으로 사용한다.
10. 오답은 문제별이 아니라 함정 유형별로 정리한다.
11. 시험에서는 쉬운 단답형과 SQL을 먼저 확보한다.
12. 고난도 C 문제 하나에 과도한 시간을 사용하지 않는다.
13. 목표는 모든 문제 정복이 아니라 14문제 이상의 안정적 정답이다.
